Types Of Exception Handling

An exception is an error which disrupts the normal flow of program instructions. PL/SQL provides us the exception block which raises the exception thus helping the programmer to find out the fault and resolve it.

  • Syntax of Exception Handling

    • DECLARE
    declarations section;
    BEGIN
    executable command(s);
    EXCEPTION
    WHEN exception1 THEN
    statement1;
    WHEN exception2 THEN
    statement2;
    [WHEN others THEN]
    /* default exception handling code */
    END;

    There are two types of exceptions defined in PL/SQL

    (i) Predefined exceptions
    (ii)User defined exception.




    (i) Predefined exceptionsThese exceptions are predefined in PL/SQL which get raised WHEN certain database rule is violated.

  • Predefined exceptions are further divided into two categories:

    • Named Predefined exceptions.
    Unnamed Predefined exceptions.

    1. Named Predefined xceptions: They have a predefined name by the system like ACCESS_INTO_NULL, DUP_VAL_ON_INDEX, LOGIN_DENIED etc.

    1. NO_DATA_FOUND: It is raised WHEN a SELECT INTO statement returns no rows.

  • Example of NO_DATA_FOUND exception

    • Select* from student;

    declare
    srol number;
    begin
    select rollno into srol from student where rollno=5;
    Dbms_output.put_line(srol);
    Exception
    when NO_DATA_FOUND then
    Dbms_output.put_line('The rollno is does not exist');
    end;





    2. TOO_MANY_ROWS : It is raised WHEN a SELECT INTO statement returns more than one row.

  • Example of TOO_MANY_ROWS exception

    • DECLARE
    sname varchar(20);
    BEGIN
    SELECT name into sname from student;
    dbms_output.put_line(sname);
    EXCEPTION
    WHEN too_many_rows THEN
    dbms_output.put_line('error SELECT too many rows');
    end;





    3. ZERO_DIVIDE : it is raises exception WHEN dividing with zero.

  • Example of ZERO_DIVIDE exception

    • DECLARE
    a number :=20;
    b number :=0;
    c number;
    BEGIN
    c:=a/b;
    dbms_output.put_line('the result of division is : '||c);
    exception
    WHEN zero_divide THEN
    dbms_output.put_line('dividing by zero please check value');
    dbms_output.put_line('the value of a is : '||a);
    dbms_output.put_line('the value of b is : '||b);
    END;





    2. unamed Predefined exception : Oracle doesn’t provide name for some system exceptions called unnamed system exceptions.These exceptions don’t occur frequently.These exceptions have two parts code and an associated message.The way to handle to these exceptions is to assign name to them using Pragma EXCEPTION_INIT.

  • Syntax of unamed Predefined exception

    • PRAGMA EXCEPTION_INIT(exception_name, -error_code);


    (ii) User defined exception : These exceptions are predefined in PL/SQL which get raised WHEN certain database rule is violated.

  • Syntax of user defined exception

    • DECLARE
    exception_name EXCEPTION;
    BEGIN
    statement(s);
    IF condition THEN
    RAISE exception_name;
    END IF;
    EXCEPTION
    WHEN exception_name THEN
    User defined statement ;
    END;


  • Example of user defined exception

    • declare
    x number ;
    e1 exception;
    begin
    x := 100;
    if (x = 100) then
    raise e1;
    end if;
    exception
    when e1 then
    dbms_output.put_line('the x value is equal to 100');
    end;
