Trigger

A database trigger is a stored program which is automatically fired or executed when some events occur. A trigger can execute in response to any of the following events:
A database manipulation (DML) statement like DELETE, INSERT or UPDATE.
A database definition (DDL) statement like CREATE, ALTER or DROP.

Parts of Triggers

A database trigger has two parts :
(i) Triggering Event
(ii) Trigger Action

When the event occurs, the database trigger fires and the PL/SQL block perfrom the action. Database trigger fire with the privilege of the owner, not the current user Therefore the owner must have appropriate access to all objects referenced by the trigger action.

(i) Triggering Event : A triggering event or statement is in the SQL statement that causes a trigger to be fired. A triggering event can be an INSERT, UPDATE, or DELETE statement on a table.

(ii) Trigger Action : A trigger action is the procedure (PL/SQL block) that contains the sql statement and PL/SQL code to be excuted when a triggering statement is issued and the trigger restriction evaluates to TRUE.


  • Syntax of Trigger

    • CREATE [OR REPLACE] TRIGGER trigger_name
    [Before | After]
    INSERT [OR] UPDATE [OR] DELETE
    ON table_name
    For each row
    DECLARE
    variable declaration;
    BEGIN
    statement trigger action
    END;

  • Example of Trigger

    • create table product(P_id number, p_name varchar(20), price number);

    create or replace trigger t1
    before insert or update on product
    for each row
    begin
    dbms_output.put_line('new p_id : ' ||:new.p_id);
    end;


    insert into product(p_id) values (101);
    update product set p_id=144 where p_id=101;

    • DROP TRIGGER trigger_name;

  • Example of drop a trigger
