Exception Handling

In PL/SQL, a waring or error condition is called an exception, which arises during program executiopn, Run-time error or exception arise from design faults, coding mistakes, hardware failures, and many other sources. when an error occurs, an exception is raised, i.e the normal exceution stops and the control transfers to the exception handling part of the PL/SQL block.

Exception are identifers in pl/sql which many be raised diring the excution of a block to teriminate its main body of actions, A block will always terminate when an exception is raised, but you many specify an 'Exception handler' to perfrom final action before the block terminates.


Advantages of PLSQL Exception Handling

Using exceptions, we can handle errors accurately with a single exception handler

When you add an exception handler to our PL/SQL block, there is no need to check for error at each point. If the exception occurs, it will be handled.

If you want to test for errors at a certain point, then we can give a single statement or a group of statements in the BEGIN-END block, with its own exception handler. By this, we can make testing as accurate we want.
